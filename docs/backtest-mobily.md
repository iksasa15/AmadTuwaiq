# Backtest: موبايلي 2014

## الحالة

**اتحاد اتصالات (موبايلي)** — `7020.SR`

في **نوفمبر 2014** أعلنت الشركة عن أخطاء محاسبية في إثبات الإيرادات (عقود broadband بالجملة وحملات ترويجية)، وأعادت إصدار أرباح 2013 والنصف الأول 2014 بخفض **1.43 مليار ريال** (Reuters، إعلانات تداول).

الذمم المدينة انتفخت بينما الأرباح المعلنة بقيت قوية — نمط كلاسيكي لـ Beneish (DSRI، TATA، CFO/NI).

## البيانات

ملف `data/backtest/mobily_pre_restatement.csv` — أرقام **مُعاد بناؤها** من السرد العام (ليست نسخة من 10-K):

| السنة | السرد |
|-------|--------|
| 2011–2013 | نمو ذمم أسرع من المبيعات |
| 2012–2013 | CFO/NI < 0.5 سنتين متتاليتين |
| 2013 | M-Score > -1.78 |

## النتائج

```bash
python -m src.backtest.mobily
pytest tests/test_backtest_mobily.py -v
```

| السنة | الدرجة | المستوى | M-Score | إشارات |
|-------|--------|---------|---------|--------|
| 2011 | ~30 | medium | -2.05 | ذمم > مبيعات |
| 2012 | ~44 | medium | -1.81 | + أرباح بلا تدفق |
| **2013** | **~57** | **high** | **-1.75** | M-Score + TATA + ذمم |

### شريحة العرض

> **«سنة قبل الإعلان (2013): درجة 57 — مرتفع — مع 4 إشارات حمراء»**

النظام **لا يتنبأ بالاحتيال** — يرفع إشارة مبكرة تستحق مراجعة قبل إعلان السوق.

## لماذا لا 76+ (critical)؟

- ML (IF/XGB) مدرّب على بيانات حالية — لا يُطبّق retroactively على 2013
- الأوزان معايرة لتجنب false positives على السوق السعودي اليوم
- **57 high** كافية للعرض: «كان سيرفع علمًا قبل سنة»
