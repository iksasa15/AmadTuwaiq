# دليل منصة رقيب للمطور

> **رقيب · Raqeeb** — منصة رقابة مالية استباقية  
> هاكاثون **امد 2026** · مصرف الإنماء × أكاديمية طويق  
> آخر تحديث: يوليو 2026

---

## 1. مقدمة

### ما هو رقيب؟

رقيب منصة ل**كشف مخاطر التلاعب المحاسبي** في الشركات المدرجة في السوق السعودي **قبل** أن يرصدها السوق أو الإعلام.

### الهدف

- حساب درجة مخاطر مركّبة (0–100) لكل شركة
- إطلاق **إشارات حمراء** عند مؤشرات تستدعي تدقيقاً إضافياً
- تقديم **أدوات عملية** لموظفة البنك (فحص محفظة، أسئلة مدقق)
- **إثبات النموذج** على أزمات حقيقية (موبايلي 2014)

### جمهوران للمنصة

| الجمهور | الاستخدام |
|---------|-----------|
| **موظفة البنك** (ائتمان/رقابة) | متابعة يومية: من أتابع؟ ماذا أسأل؟ ماذا أفعل؟ |
| **الحكام / العرض** | إثبات موبايلي + درع الإنماء + رؤية مستقبلية |

## 2. تشغيل المشروع

### Backend

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m src.ingestion.run_all --skip-fetch
python -m src.models.run_beneish
uvicorn src.api.main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

### روابط

| الخدمة | الرابط |
|--------|--------|
| Swagger API | http://localhost:8000/docs |
| لوحة التحكم | http://localhost:5173 |

### وضع الديمو vs وضع API

- **شريط علوي** (`DemoBanner`) يبدّل بين الوضعين
- **وضع الديمو (افتراضي):** بيانات غنية من ملفات `frontend/src/data/*` — مناسب للعرض بدون backend
- **وضع API:** يتصل بـ FastAPI على `VITE_API_URL` (افتراضي `http://localhost:8000/api/v1`)
- المنطق في: `frontend/src/api/dataSource.ts` + `frontend/src/hooks/useDemoMode.tsx`

## 3. المعمارية (مختصر)

```
yfinance ──► ETL ──► features.parquet
                          │
                    ML + Beneish (8 مؤشرات)
                          │
                   scores.parquet + SQLite
                          │
                     FastAPI ──► React (RTL)
```

### معادلة الدرجة المركّبة

```
Risk = 0.40×M-Score + 0.20×IsolationForest + 0.15×XGBoost + 0.25×Rules
```

| المكوّن | الوزن | المصدر |
|---------|-------|--------|
| M-Score (Beneish) | 40% | تلاعب محاسبي كلاسيكي |
| Isolation Forest | 20% | شذوذ قطاعي |
| XGBoost | 15% | احتمال تلاعب |
| قواعد الإشارات | 25% | CFO/NI، ذمم، TATA، DEPI… |

### مستويات المخاطر

| النطاق | المستوى |
|--------|---------|
| 0–25 | منخفض |
| 26–50 | متوسط |
| 51–75 | مرتفع |
| 76–100 | حرج |

## 5. تفصيل كل صفحة

### 5.1 لوحة التحكم (`home`)

**الفكرة:** نقطة البداية — نظرة عامة على السوق والشركات الأخطر.

**المكوّنات:**
- `PageIntro` — يشرح الفائدة والمحتوى
- `QuickStartCards` — 3 بطاقات: ابدئي المتابعة · أثبتي النموذج · افحصي محفظة
- `DemoBanner` — تبديل ديمو/API
- بطاقة ملخص: عدد الشركات + متوسط المخاطر + `RiskDonut`
- شريط أعلى 5 مخاطر (قابل للنقر → ملف شركة)
- `ActivityFeed` — آخر النشاط (**ديمو فقط**)
- جدول الشركات + فلتر قطاع + ترتيب بالخطر

**مصدر البيانات:**
- API: `GET /api/v1/companies` + `GET /api/v1/market/overview`
- ديمو: `frontend/src/data/demo.ts`

---

### 5.2 الإشارات (`alerts`)

**الفكرة:** مركز الإشارات الحمراء — مرتبة حسب الخطورة.

**المكوّنات:**
- إحصائيات: إجمالي · حرجة · تحذيرية · معلوماتية
- `FlagCard` لكل إشارة (مع سؤال مدقق + نسخ)
- رابط للشركة من كل إشارة

**مصدر البيانات:**
- ديمو: `DEMO_ALERTS` في `frontend/src/api/dataSource.ts`
- API: `GET /api/v1/companies/{ticker}/flags`

**ملاحظة:** لا يوجد `ActivityFeed` هنا (موجود في لوحة التحكم فقط).

---

### 5.3 القطاعات (`sectors`)

**الفكرة:** تحليل مخاطر على مستوى الصناعة.

**المكوّنات:**
- رسم أعمدة: متوسط درجة المخاطر بالقطاع
- بطاقات قطاع: درجة، عدد شركات، وصف، محرّك الخطر

**مصدر البيانات:**
- `overview.sector_breakdown` + `frontend/src/data/demoExtras.ts` (DEMO_SECTOR_META)

---

### 5.4 درع الإنماء (`portfolio`)

**الفكرة:** ماسح المحفظة الائتمانية — أداة البنك الأساسية.

**المكوّنات:**
- رفع CSV/XLSX (سحب وإفلات)
- زر «فحص ملف الإنماء (ديمو)»
- تحميل ملف Excel: `frontend/public/demo/alinma_corporate_portfolio_2025.xlsx`
- تقرير: نسبة أمان المحفظة + جدول الشركات + تصدير CSV

**مصدر البيانات:**
- API: `POST /api/v1/portfolio/scan`
- ديمو: `DEMO_ALINMA_PORTFOLIO` في `frontend/src/data/strategicDemo.ts`

---

### 5.5 أسئلة المدقق (`prompts`)

**الفكرة:** أسئلة استجواب جاهزة لمواجهة الشركة في مقابلة التمويل.

**المكوّنات:**
- بطاقات لكل نوع إشارة: عنوان + أسلوب تفاوض + قائمة أسئلة

**مصدر البيانات:**
- ديمو: `DEMO_AUDITOR_PROMPTS` في `strategicDemo.ts`
- API: حقل `interrogation_prompt_ar` في استجابة `/flags` (يظهر أيضاً في `FlagCard`)

---

### 5.6 إثبات النموذج (`backtest`)

**الفكرة:** هل رقيب كان سيعطي إنذاراً **قبل** الإعلان الرسمي؟ — للعرض على الحكام.

**المكوّنات:**
- تبديل: موبايلي 2014 / المعجل 2016
- إحصائيات: درجة الذروة · أشهر قبل الكشف · تاريخ الكشف
- `InteractiveTimeline` — خط زمني تفاعلي
- زر «فتح دراسة موبايلي» → يفتح `MobilyCasePage`

**مصدر البيانات:**
- موبايلي: `MOBILY_TIMELINE` في `dataSource.ts` + `data/known_cases.json` (API)
- المعجل: `ALMAJIL_TIMELINE` في `strategicDemo.ts`

---

### 5.7 قدرات مستقبلية (`future`)

**الفكرة:** ابتكارات لا توفرها أنظمة البنك اليوم — **ديمو / رؤية 2027 فقط**.

**التبويبات الفرعية:**

| التبويب | الاسم | المكوّن | الوصف |
|---------|-------|---------|-------|
| `sentiment` | تحليل نبرة التقارير | `SentimentRadar.tsx` | مقارنة نبرة تقرير مجلس الإدارة بالأرقام |
| `network` | شبكة العلاقات المالية | `ForensicNetwork.tsx` | رسم شبكة كيانات وتدفقات مشبوهة |
| `whatif` | محاكاة ماذا لو؟ | `WhatIfSimulator.tsx` | sliders لتغيير افتراضيات وإعادة حساب الدرجة |

**مصدر البيانات:** `frontend/src/data/strategicDemo.ts` — **لا backend**

**ملاحظة:** محاكاة «ماذا لو» **الحقيقية** (API) موجودة في **ملف الشركة** وليس هنا.

### 5.8 دليل الاستخدام (`guide`)

**الفكرة:** مسارات جاهزة للمستخدم الجديد.

**مسار العمل (~3 دقائق)** — موظفة البنك:
1. لوحة التحكم — اختاري شركة عالية الخطر
2. الإشارات — راجعي الإشارات الحرجة
3. ملف الشركة — افتحي الشركة من الجدول
4. أسئلة المدقق — جهّزي أسئلة المقابلة

**مسار العرض (~5 دقائق)** — الحكام:
1. إثبات النموذج — موبايلي قبل الكشف
2. درع الإنماء — افحصي محفظة البنك
3. قدرات مستقبلية — اعرضي الرؤية
