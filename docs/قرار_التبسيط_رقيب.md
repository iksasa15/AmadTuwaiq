# قرار التبسيط — رقيب: ما يبقى وما يُحذف
**لماذا هذا الملف:** المنصة توسّعت لـ 9 تبويبات + صفحات فرعية متعددة. هذا حجم أكبر من اللازم لعرض هاكاثون مدته 3-5 دقائق، ويرفع خطر تعطّل شي حي أثناء العرض. الهدف: تقليص العرض لأقوى 3 نقاط قوة فقط + العناصر الداعمة الضرورية لها، وحذف كل شي يشتت أو يكرر نفس الفكرة.

---

## 🏆 أقوى 3 نقاط قوة — هذي محور العرض والتطوير من الآن

اخترتها بناءً على 4 معايير: **حقيقية (API لا ديمو)** + **فريدة عن أي نظام بنكي حالي** + **قصة مقنعة للحكام** + **مرتبطة مباشرة بمصرف الإنماء كراعي**.

### 1️⃣ إثبات النموذج (Backtest + دراسة موبايلي)
**التبويب:** `backtest` | **الملفات:** `BacktestPage.tsx` → `TimeMachine.tsx` + `MobilyCasePage.tsx`

**ليش الأقوى:** يجاوب مباشرة على أصعب سؤال ممكن يجيكم من الحكام: *"كيف نثبت أن نظامكم شغّال فعليًا؟"* — إثبات تاريخي حقيقي (API + `known_cases.json`) أقوى من أي ادعاء نظري.

### 2️⃣ درع الإنماء (فحص المحفظة الائتمانية)
**التبويب:** `portfolio` | **الملفات:** `PortfolioPage.tsx` → `PortfolioScanner.tsx`

**ليش الأقوى:** الوحيدة اللي تربط المشروع مباشرة بمصرف الإنماء (الراعي = العميل المستهدف). تحوّل المشروع من "فكرة طلابية" إلى "أداة نقدر نبيعها للبنك بكرة". API حقيقي بالكامل.

### 3️⃣ محاكاة "ماذا لو؟" — النسخة الحقيقية فقط (داخل ملف الشركة)
**الموقع:** تبويب "محاكاة ماذا لو" داخل `CompanyPage.tsx` | **الملف:** `CompanyWhatIfSimulator.tsx`

**ليش الأقوى:** أعلى نسبة "بهرجة أمام الحكام / جهد تطوير" بكل المنصة — تحريك Slider والدرجة تتغير حيّة أمامهم مباشرة من API حقيقي، بدون أي ادعاء ديمو.

> ⚠️ **مهم:** فيه نسخة ثانية مكررة من "ماذا لو" داخل تبويب "قدرات مستقبلية" (ديمو فقط) — هذي التكرار بالذات سبب رئيسي في اللخبطة، واتحذف بالكامل (انظر قسم الحذف).

## ✅ يبقى — لكنه دعم وليس محور العرض

| العنصر | الحالة | السبب |
|---|---|---|
| **لوحة التحكم** (`home`) | يبقى، لكن يُبسّط | نقطة الدخول الإلزامية لأي عرض أو استخدام يومي — لا غنى عنها |
| **ملف الشركة** (`CompanyPage`) | يبقى كامل | يحوي نقطة القوة #3 + كل تفاصيل الشركة، هو "القلب" التقني للمنصة |
| **الإشارات الحمراء داخل `FlagCard`** (مع سؤال المدقق والنسخ) | يبقى، لكن **بدون تبويب Sidebar مستقل** | الميزة نفسها (سؤال الاستجواب) قيّمة وتبقى، لكن مكانها الطبيعي هو بطاقة الإشارة داخل ملف الشركة، مو صفحة منفصلة |

## 🗑️ يُحذف بالكامل — واضح ومباشر للمطور

### 1. تبويب «قدرات مستقبلية» (`future`) — **احذف التبويب بالكامل**
**السبب الرئيسي:** كل محتواه ديمو فقط (لا API)، وفيه تكرار مباشر لنقطة القوة #3.

يشمل الحذف:
- الملف: `FutureCapabilitiesPage.tsx`
- المكوّن: `SentimentRadar.tsx` (تحليل نبرة التقارير) — فكرة قوية مفاهيميًا لكن غير منجزة فعليًا (ديمو فقط)، عرضها كـ"قدرة مستقبلية" غير مؤكدة يفتح باب أسئلة يصعب الدفاع عنها
- المكوّن: `ForensicNetwork.tsx` (شبكة العلاقات المالية) — نفس السبب: بيانات غير موثوقة، ديمو فقط
- المكوّن المكرر: نسخة `WhatIfSimulator.tsx` هذي (غير نسخة ملف الشركة الحقيقية) — **تكرار كامل** لنقطة القوة #3 بنسخة ديمو أضعف، يسبب لخبطة "أي وحدة الحقيقية؟"
- إزالة الإدخال من `frontend/src/config/navigation.ts` (حذف `TabId: 'future'` و `PAGE_META` المرتبط)
- إزالة route من `App.tsx`
- ملف البيانات `frontend/src/data/strategicDemo.ts`: احذف منه فقط أقسام (`DEMO_SENTIMENT`, `DEMO_NETWORK`, أي what-if مكرر) — **احتفظ** بباقي الملف لأنه يحوي أيضًا `DEMO_ALINMA_PORTFOLIO` و `DEMO_AUDITOR_PROMPTS` و `ALMAJIL_TIMELINE` المستخدمة بنقاط القوة المُبقاة

### 2. تبويب «القطاعات» (`sectors`) — **احذف التبويب المستقل**
**السبب:** تحليل جيد لكن ثانوي، ما يخدم أي من نقاط القوة الثلاث مباشرة، ويضيف تبويب كامل لقيمة هامشية.
- الملف: `SectorsPage.tsx`
- إزالة `TabId: 'sectors'` من `navigation.ts` و route من `App.tsx`
- **بديل اختياري (لو بغيتوا الاحتفاظ بشي بسيط منه):** انقلوا رسم "متوسط المخاطر بالقطاع" كبطاقة واحدة صغيرة داخل لوحة التحكم بدل صفحة كاملة — قرار المطور حسب الوقت المتبقي

### 3. تبويب «أسئلة المدقق» المستقل (`prompts`) — **احذف الصفحة، أبقِ الميزة**
**السبب:** الميزة نفسها (`interrogation_prompt_ar`) موجودة أصلًا وتُعرض داخل `FlagCard` بكل مكان بالمنصة (الإشارات + ملف الشركة). تخصيص تبويب Sidebar كامل لنفس المحتوى المعروض بمكان ثاني = تكرار بلا داعٍ.
- الملف: `PromptsPage.tsx` → `AuditorPrompts.tsx` — **احذف**
- إزالة `TabId: 'prompts'` من `navigation.ts` و route من `App.tsx`
- **لا تلمس** `interrogation_prompt_ar` بالـ API ولا منطق `FlagCard.tsx` — هذا يبقى شغّال بمكانه الحالي

### 4. `ActivityFeed` في لوحة التحكم — **احذف المكوّن**
**السبب:** عنصر ديمو تجميلي بدون قيمة حقيقية أو ارتباط بنقاط القوة، مجرد تشتيت بصري في أهم صفحة (لوحة التحكم).
- احذف استدعاء `ActivityFeed` من `MarketOverview.tsx`
- الملف نفسه `ActivityFeed.tsx` يقدر المطور يحذفه أو يتركه غير مستخدم (تنظيف اختياري)

### 5. تبويبا «دليل الاستخدام» (`guide`) و«عن رقيب» (`about`) — **احذف من الـ Sidebar**
**السبب:** محتوى وصفي/تعريفي بدون بيانات أو تفاعل حقيقي، مفيد لمستخدم حقيقي بعد الإطلاق وليس لعرض هاكاثون 3-5 دقائق.
- احذف `GuidePage.tsx` و `AboutPage.tsx` من الـ Sidebar (`navigation.ts` + `App.tsx`)
- **بديل مقترح:** لو الفريق يبي يحتفظ بمحتوى "عن رقيب" (الشعار + الفكرة)، حطّوه كزر واحد بسيط بالهيدر يفتح نافذة صغيرة (Modal)، بدل صفحة/تبويب كامل بالـ Sidebar

## 📋 خريطة التنقل النهائية بعد التبسيط

```
المراقبة
  └── لوحة التحكم (home)          ← نقطة الدخول

أدوات العمل
  └── درع الإنماء (portfolio)      ← نقطة القوة #2

الإثبات والعرض
  └── إثبات النموذج (backtest)    ← نقطة القوة #1
```

**صفحة فرعية (بدون تبويب Sidebar):**
```
ملف الشركة (CompanyPage)
  ├── نظرة عامة
  ├── محاكاة ماذا لو            ← نقطة القوة #3
  ├── القوائم المالية
  └── الإشارات (FlagCard + سؤال المدقق)
```

**من 9 تبويبات إلى 3 تبويبات + صفحة شركة واحدة.** هذا يعطي عرضًا مركّزًا 3 نقاط فقط، وسطح أصغر بكثير لأي عطل تقني حي.
