```
┌─────────────────────────────────────────────────────────┐
│                     طبقة البيانات                        │
│  Scraper (تداول/أرقام) → Raw Store → ETL → Feature DB   │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│                    طبقة الذكاء                           │

                           │
┌──────────────────────────▼──────────────────────────────┐

└─────────────────────────────────────────────────────────┘
```

**المخرج النهائي للجنة التحكيم:** لوحة تحكم حيّة تعرض درجة مخاطر احتيال لـ 30+ شركة سعودية حقيقية، مع تفسير عربي واضح لكل إشارة حمراء.

## المكدّس التقني (Tech Stack) — قرارات نهائية لا تعيد التفكير فيها

| الطبقة | الاختيار | السبب |
|---|---|---|
| اللغة | Python 3.11 | نظام بيئي ML كامل |
| جلب البيانات | `requests` + `BeautifulSoup` + `yfinance` | كافي، لا تحتاج Selenium إلا للضرورة |
| معالجة | `pandas` + `numpy` | معيار الصناعة |
| النموذج | `scikit-learn` (IsolationForest) + `xgboost` | سريع التدريب، قابل للتفسير |
| التفسير | `shap` | لجنة التحكيم ستسأل "ليش هالدرجة؟" — هذا جوابك |
| API | `FastAPI` + `uvicorn` | توثيق تلقائي /docs يبهر الحكّام |
| قاعدة بيانات | `SQLite` (لا PostgreSQL!) | مشروع هاكاثون، لا تعقّد |
| الواجهة | React + Vite + Tailwind + Recharts | سريع، RTL سهل |
| النشر | Docker Compose + خدمة مجانية (Railway/Render) | رابط حي > localhost |
| إدارة الكود | GitHub + branches (main/dev) | الفريق يشتغل بالتوازي |

> ⚠️ **قاعدة ذهبية:** أي تقنية غير مذكورة فوق = تشتيت. لا Kubernetes، لا Kafka، لا LLM fine-tuning. الهدف Demo يشتغل، مو بنية Google.

### الساعات 9–11: دراسة Beneish M-Score (نظري + كود أولي)
النموذج = 8 نسب مالية. اكتب `src/models/beneish.py` بالمعادلات:

| المؤشر | المعادلة | يكشف |
|---|---|---|
| DSRI | (ذمم/مبيعات)ₜ ÷ (ذمم/مبيعات)ₜ₋₁ | تضخيم إيرادات وهمية |
| GMI | هامش ربحₜ₋₁ ÷ هامش ربحₜ | تدهور يدفع للتلاعب |
| AQI | جودة الأصول | رسملة مصاريف مشبوهة |
| SGI | نمو المبيعات | نمو غير طبيعي |
| DEPI | معدل الإهلاك | إبطاء إهلاك لتضخيم ربح |
| SGAI | مصاريف بيعية/مبيعات | — |
| LVGI | الرافعة المالية | ضغط ديون |
| TATA | Total Accruals/Assets | **الأقوى** — أرباح ورقية بلا كاش |

```
M = -4.84 + 0.92·DSRI + 0.528·GMI + 0.404·AQI + 0.892·SGI
    + 0.115·DEPI - 0.172·SGAI + 4.679·TATA - 0.327·LVGI

M > -1.78  ⇒  احتمال تلاعب مرتفع
```

`src/models/scoring.py` — لا تعتمد على M-Score وحده:
```
risk_score = w1·norm(M-Score) + w2·norm(anomaly_score) + w3·rule_flags_score
البداية: w = [0.5, 0.3, 0.2] — ستضبطها يوم 5
```
قواعد الإشارات الحمراء (rule flags) — كل قاعدة لها نص عربي جاهز:
| القاعدة | النص العربي للمستخدم |
|---|---|
| CFO/NI < 0.5 لسنتين | "أرباح محاسبية لا يقابلها تدفق نقدي — قد تكون أرباحًا ورقية" |
| نمو ذمم > 1.5× نمو مبيعات | "الذمم المدينة تنمو أسرع من المبيعات — احتمال إيرادات مبكرة/وهمية" |
| DEPI > 1.2 | "تباطؤ ملحوظ في الإهلاك — قد يضخّم الأرباح" |
| قفزة LVGI | "ارتفاع حاد في الرافعة المالية" |

## اليوم 4 — الأربعاء 9 يوليو: كشف الشذوذ + XGBoost
**الهدف:** طبقة ML فوق القواعد — هذا ما يميزكم عن مشروع Excel.

### الساعات 1–3: Isolation Forest (بدون بيانات موسومة!)
هذا حلّك لمشكلة "ما عندنا بيانات احتيال موسومة":
```python
from sklearn.ensemble import IsolationForest
model = IsolationForest(n_estimators=200, contamination=0.1, random_state=42)
# الميزات: الـ 8 نسب + النسب الإضافية، بعد StandardScaler
anomaly_scores = -model.score_samples(X)  # أعلى = أشذ
```
- الشركة الشاذة عن أقرانها في **نفس القطاع** أهم من الشاذة عن السوق كله → درّب نموذج لكل قطاع إذا العينة تسمح، وإلا أضف القطاع one-hot

### الساعات 4–7: بيانات تدريب اصطناعية موسومة (الحيلة الذكية)
ابنِ dataset تدريب لـ XGBoost بحقن تلاعب اصطناعي:
```python
def inject_fraud(row):
    """يحاكي أنماط تلاعب موثقة أكاديميًا"""
    r = row.copy()
    r["receivables"] *= np.random.uniform(1.4, 2.2)   # تضخيم ذمم
    r["revenue"]     *= np.random.uniform(1.1, 1.35)  # تضخيم مبيعات
    r["depreciation"] *= np.random.uniform(0.6, 0.85) # إبطاء إهلاك
    r["label"] = 1
    return r
```
- خذ الشركات النظيفة (label=0)، أنشئ نسخًا محقونة (label=1) بنسبة 1:1
- درّب XGBoost على الميزات المشتقة (النسب، ليس الأرقام الخام)
- **هذه نقطة قوية للعرض:** "درّبنا النموذج على أنماط تلاعب موثقة في الأدبيات المحاسبية"

### الساعات 8–9: تقييم صادق
- Train/test split 80/20 + 5-fold CV
- سجّل: Precision, Recall, F1, ROC-AUC — احفظها في `reports/model_eval.md`
- **مصداقية للعرض:** اذكر الرقم الحقيقي حتى لو 0.85 وليس 0.99 — الحكام التقنيون يشمّون الأرقام المزيفة

### الساعات 10–12: SHAP للتفسير
```python
import shap
explainer = shap.TreeExplainer(xgb_model)
shap_values = explainer.shap_values(X)
```
- لكل شركة: استخرج أعلى 3 ميزات دفعت الدرجة للأعلى → حوّلها لجُمل عربية عبر قاموس `feature → template_ar`
- احفظ صورة SHAP summary plot في `reports/` — ستدخل العرض التقديمي
- حدّث الدرجة المركّبة بدمج مخرجات النموذجين

## 1) الإعداد والقرارات التقنية


| القرار | الاختيار | السبب |
|---|---|---|
| اللغة | TypeScript | الـ API Contract يتحول لـ types — أخطاء الدمج تنكشف قبل التشغيل |

## 4) الأنواع (types.ts) — انسخها من الـ Contract حرفيًا

```ts
export type RiskLevel = 'low' | 'medium' | 'high';

export interface CompanySummary {
  ticker: string;
  name_ar: string;
  sector: string;
  risk_score: number;        // 0–100
  risk_level: RiskLevel;
  score_delta: number;       // التغير عن الربع السابق
}

export interface Flag {
  flag_id: string;
  title_ar: string;          // "الذمم تنمو أسرع من المبيعات"
  severity: 1 | 2 | 3;
  explanation_ar: string;    // جملتان بالعربي
  evidence: {                // ⭐ الدليل الرقمي — قلب المصداقية
    metric_ar: string;       // "نمو الذمم المدينة"
    value: string;           // "84%"
    benchmark: string;       // "نمو المبيعات: 12%"
  };
}

export interface CompanyDetail extends CompanySummary {
  score_history: { period: string; score: number }[];
  beneish: Record<string, { value: number; sector_avg: number; threshold: number }>;
  flags: Flag[];
  confidence: 'high' | 'low';   // low = بيانات ناقصة
}
```

### RiskGauge — التوقيع البصري للمشروع
SVG يدوي (لا مكتبة): قوس 270° بتدرج أخضر→برتقالي→أحمر، إبرة تتحرك بـ CSS transition عند التحميل (أنيميشن واحدة مقصودة — الرقم "يصعد" من 0 للدرجة خلال ثانية). الرقم بالمنتصف 64px عريض بلون المستوى، وتحته الشارة العربية. هذا المكوّن سيظهر في كل لقطة شاشة وفي العرض التقديمي — استثمر فيه.

### FlagCard — قلب المصداقية
- حد ابتدائي (border-inline-start) بلون الخطورة بسماكة 4px
- الخطورة = عدد أيقونات ⚠️ (1–3) وليس نصًا — يُقرأ بلمحة
- **قسم الدليل** أسفل البطاقة بخلفية رمادية فاتحة وخط أصغر: `المقياس: القيمة | المرجع` — هذا ما يفرّق بين "ذكاء اصطناعي يتهم" و"نظام يبرهن"

## 9) فخاخ RTL الشائعة — راجعها قبل كل commit

1. Recharts لا يعكس المحاور تلقائيًا: مرّر `reversed` لمحور X في الرسم الخطي حتى يمشي الزمن يمين→يسار
2. أيقونات الأسهم (←/→) لا تنعكس مع dir — اختر الأيقونة الصحيحة يدويًا
3. الأرقام مع النص العربي: غلّف الأرقام المختلطة بـ `<span dir="ltr">` عند الحاجة (مثل "2050.SR")
4. `space-x-*` في Tailwind لا يدعم RTL قبل v3.3 — استخدم `gap` مع flex دائمًا
5. اختبر بتصغير الشاشة لـ 375px: الجدول يتحول لبطاقات مكدسة (الحكام يفتحون من الجوال)
